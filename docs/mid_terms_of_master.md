## 1. 课题简介

### 1.1 课题背景

企业在产品设计与运营过程中积累了大量技术文档，涵盖产品说明书、技术白皮书、规格文档与操作指南等多种形式，蕴含着组件构成、性能参数、安全属性等关键质量信息。然而，传统质量管理方法难以从这类非结构化文档中自动发现潜在质量影响因素，人工阅读效率低下且无法规模化应用 [1][25]。这些产品文档通常以PDF格式呈现，兼具多模态（文本、表格、示意图、插图）和长文档特征，同一产品组件的完整信息往往跨越多个章节和页面，且产品知识遵循"产品—组件—功能—参数—属性"的层次语义结构，给自动化信息挖掘带来了独特的技术挑战。本课题来源于学校与企业的合作项目，面向消费电子领域的产品质量管理需求，研究如何从多模态产品文档中自动挖掘并结构化产品质量关键因素，为企业提供智能化的知识检索与问答支持。

近年来，知识图谱与大语言模型（LLM）的快速发展为上述问题提供了新的解决思路。知识图谱以图结构显式建模实体间语义关系，天然适合表达产品层次化知识体系，并可通过关系路径支持多跳推理 [25][26][27]。检索增强生成（RAG）范式将知识图谱的结构化检索能力与LLM的生成能力相结合，成为构建知识问答系统的主流框架 [22]。GraphRAG [4]、LightRAG [5] 等图结构RAG方法及RAG-Anything [17] 等多模态统一RAG框架，使从复杂文档中构建知识图谱并支持问答成为可能。然而，现有RAG框架应用于产品文档场景时仍存在若干不足，包括分块级图谱构建无法捕获跨页面的产品级结构化知识、领域知识抽取缺乏可复用的模块化封装机制、单次"检索-生成"管道缺乏针对不同问题类型的策略支持，以及生成阶段缺乏事实正确性的独立验证机制。上述不足构成了本文研究的直接出发点。


### 1.2 主要研究内容和目标

本文面向多模态产品文档中产品质量关键因素的自动挖掘与问答这一核心任务，在RAG-Anything多模态统一RAG框架基础上，从知识图谱增强构建与检索生成优化两个层面提出针对性改进，构建PRAG（Product Retrieval-Augmented Generation）框架，主要研究内容如下。

基于领域知识驱动的知识图谱增强构建方法（PRAG）。提出将领域先验知识以"技能"形式结构化定义，每个技能包含目标输出结构定义（Schema）、抽取步骤声明（SKILL.md）和提示模板（Prompts）三类文件，形成可跨文档复用的知识抽取规格。领域知识抽取智能体（DKEAgent）识别文档所属领域后激活对应技能，在基础图谱上执行全局结构化抽取。其中，一对一关系字段由单个子Agent整体输出，一对多关系字段则先枚举全部实体，再为每个实体独立并行抽取。全部结果以程序逻辑合并，经语义实体对齐融合进基础图谱，形成增强图谱 $G_{v2}$。该机制解决了传统分块抽取无法建立跨页面语义关联的问题。

基于迭代验证反馈的自适应多智能体检索增强生成方法（A-PRAG）。在PRAG基础上提出多智能体闭环问答架构。系统首先对用户问题进行类型判定（事实型、计数型、视觉型、列举型、不可回答型），据此为检索智能体注入差异化检索策略；检索智能体生成草稿回答后，独立验证智能体从证据充分性、完整性、可回答性与准确性四个维度进行交叉核查；验证不通过时，生成指向具体问题的结构化反馈驱动检索智能体定向重检索，直至验证通过或达到最大迭代次数。全部流程控制逻辑以确定性程序实现，将传统开环生成流程升级为具备自纠错能力的多智能体闭环系统。

研究目标是在多模态产品文档问答基准 MMLongBench-Doc（Guidebooks子集，23篇，196问答对）[23] 和 MPMQA（PM209子集，45篇，4,830问答对）[24] 上，实现相比RAG-Anything基线方法的显著性能提升，并通过消融实验验证各模块的独立贡献及其协同增益。

### 2.2 实际进展情况

论文实际工作经历了文献调研、技术路线调整、核心方法研发与实验验证四个阶段，较开题报告原定计划在研究周期与技术路线上均有所调整。

（一）文献调研与技术路线评估（2023 年 9 月—2024 年 3 月）

按照开题报告计划，前期开展了较为深入的文献调研，重点梳理了知识图谱构建、多模态信息处理、信息抽取与问答系统等方向的研究现状。调研发现，随着大语言模型（LLM）技术的快速迭代，以 ChatGPT、GPT-4 为代表的通用大模型已在零样本条件下展现出接近甚至超越传统有监督方法的信息抽取与推理能力。GraphRAG [4]、LightRAG [5] 等图结构检索增强生成（RAG）框架相继出现，直接从文档构建知识图谱并支持问答的端到端范式迅速成为主流。与此同时，RAG-Anything [17] 等多模态统一RAG框架进一步将这一范式拓展至PDF等富格式文档，使多模态知识图谱构建和问答具备了实用化基础。

基于此，课题组重新审视了原定技术路线。原开题方案以知识图谱嵌入（表示学习）为核心，但实际推进中暴露出明显制约。传统嵌入方法的训练需要大量标注三元组数据，企业场景中产品文档缺乏此类资源；嵌入向量的相似度检索也难以支持多跳复杂推理；而LLM能力提升后，基于提示工程的RAG范式在同等信息源条件下已可达到更高的问答准确率，且无需额外训练。经导师和企业合导师共同讨论，决定将技术路线调整为"基于大语言模型的知识图谱自动构建与检索增强生成"方向，更符合实际应用约束，也与领域技术发展方向一致。这一调整并非推翻前期积累，调研阶段对知识图谱结构设计的理解在后续技能库Schema设计中得到了直接运用。

（二）基础方法构建与问题发现（2024 年 3 月—2024 年 9 月）

技术路线确定后，以 RAG-Anything 为基础框架建立了性能基线，在 MMLongBench-Doc（Guidebooks 子集）[23] 和 MPMQA（PM209 子集）[24] 两个多模态产品文档问答数据集上完成了初步评测。基线实验揭示了现有框架在产品文档场景下的两类核心缺陷：

其一，知识图谱构建层面存在跨页面信息断层问题。RAG-Anything 采用通用分块级实体关系抽取策略，受限于单块上下文窗口，同一产品（如智能手表）的组件、参数、安全特性等信息散布于文档不同章节，无法被整合到一个完整的产品知识条目中，导致图谱中产品级结构化知识严重缺失。

与此相关的是检索生成层面的开环管道局限。单次"检索-生成"管道对不同类型的产品质量问题（如计数型、列举型、视觉型）缺乏针对性策略，且生成结果不经事实核查即直接输出，在检索结果存在噪声或缺失时容易产生幻觉性回答。

（三）核心方法研发（2024 年 9 月—2025 年 6 月）

针对上述两类问题，分两阶段研发了本文的核心方法。

第一阶段研发了 PRAG（Product Retrieval-Augmented Generation）框架，重点解决知识图谱构建层面的缺陷。设计了声明式技能库（Skill Library），将产品领域知识以结构化文件封装为可跨文档复用的知识抽取规格（详见3.1节）。在此基础上实现了领域知识抽取智能体（DKEAgent），采用"枚举实体—并行抽取—程序合并—语义对齐融合"的全局抽取流程，克服了分块抽取的上下文局限，将各章节分散的产品层次化知识完整聚合进基础图谱，形成增强图谱 $G_{v2}$。

第二阶段研发了 A-PRAG（Agentic PRAG）框架，重点解决检索生成层面的缺陷。该框架引入问题类型分类机制，将问题划分为事实型、计数型、视觉型、列举型和不可回答型五类，为检索智能体注入差异化检索策略。同时引入独立验证智能体，从证据充分性、完整性、可回答性与准确性四个维度对草稿回答进行交叉核查。验证不通过时，系统生成结构化反馈驱动检索智能体定向重检索，将整个问答流程升级为具备自纠错能力的多智能体闭环系统。

（四）实验验证与论文撰写（2025 年 6 月—至今）

在 MMLongBench-Doc 和 MPMQA 两个数据集上开展了完整的对比实验、消融实验与案例分析。主要实验结果如表 2 所示。

表 2 主要实验结果（平均准确率）

| RAG-Anything（基线） | 41.2 | 39.6 | 40.4% | — |

## 3. 论文工作成果介绍

### 3.1 课题所实施的解决方案介绍

本文在 RAG-Anything 多模态统一 RAG 框架基础上，提出 PRAG 与 A-PRAG 两类方法，从知识存储增强与检索生成优化两个层面解决多模态产品文档问答任务中的核心问题。

系统整体架构如下。系统以多模态产品文档（PDF）为输入，经 MinerU 解析后提取文本、图片、表格等多模态内容，利用 VLM 对非文本内容进行语义编码，通过 LightRAG 底层引擎完成初始知识图谱（基础图谱 $G_{v1}$）构建。在此基础上，PRAG 引入领域知识驱动的增强构建机制，生成结构化程度更高的增强图谱 $G_{v2}$；A-PRAG 在图谱之上引入多智能体闭环问答架构。两种方法层次清晰，可独立或组合使用。

#### 3.1.1 基于领域知识驱动的知识图谱增强构建方法

PRAG 的核心设计是声明式技能库（Skill Library）与领域知识抽取智能体（DKEAgent）。技能库将领域先验知识以模块化"技能"形式封装，每个技能由三类声明式文件组成。`schema.json` 定义目标输出结构，字典型字段对应一对一关系，列表型字段对应一对多关系；`SKILL.md` 声明抽取步骤，规定步骤顺序与字段绑定；`prompts/` 存放各步骤的自然语言约束模板。三类文件分别回答"抽取什么""按何顺序抽取""如何引导LLM"，实现领域知识定义与执行逻辑的完全解耦，扩展至新领域仅需编写新的技能定义文件。

DKEAgent 分两阶段运行。在领域识别与技能激活阶段，主智能体对基础图谱发起语义检索，读取技能库中各技能的领域描述，由 LLM 推断文档领域并激活匹配技能。在基于 Schema 的结构化知识提取阶段，一对一关系字段由单个子 Agent 完成全局检索与整体结构化输出；一对多关系字段分枚举和抽取两步，枚举子 Agent 先在图谱全局范围内列举所有实体名称，再为每个实体独立创建子 Agent 并行执行结构化抽取；全部步骤完成后以纯程序逻辑合并，输出领域知识集 $I_D$。这种"程序编排 + LLM 叶节点"的分层架构，使步骤顺序、并行调度和结果合并由确定性程序控制，LLM 仅在叶节点的实际检索与抽取子任务中发挥作用。

领域知识集 $I_D$ 经两阶段融合写入基础图谱。知识映射阶段将 $I_D$ 的五个概念域（产品、组件、功能、参数、属性）转换为图谱节点与有向边，形成以产品节点为根的层次化结构。语义实体对齐阶段对每个待写入节点构造查询（名称与描述拼接），在基础图谱实体向量空间检索最近邻，余弦相似度超过阈值 $\tau$ 时归并至已有节点，否则创建新节点，最终形成增强图谱 $G_{v2}$。

以 HUAWEI WATCH D 说明书（64 页）为例，增强前后节点数几乎未变（由1,318增至1,322），关系数增加 76 条（由3,445增至3,521），核心产品节点关联数从 20 增至 41（+105%）。新增组件（Airbag、Battery 等）和功能（Sleep Monitoring、SpO₂ Measurement 等）均来自文档不同章节的跨页面聚合。

#### 3.1.2 基于迭代验证反馈的自适应多智能体检索增强生成方法

A-PRAG 在增强图谱 $G_{v2}$ 之上引入多智能体闭环架构，由检索 Agent、验证 Agent 与确定性调度逻辑三部分构成。

在问题分类与策略注入方面，系统将用户问题分为五类（事实型、计数型、视觉型、列举型、不可回答型），并为每类注入差异化检索策略。事实型优先精确文本检索；计数型强制跨页枚举扫描，禁止依赖摘要统计；视觉型强制调用视觉语义理解，禁止无图作答；列举型要求多页扫描保证完整性；不可回答型要求充分多路检索后方可判定。分类逻辑以确定性程序实现，保证策略一致注入。

在检索 Agent 分层知识检索方面，检索 Agent 配备七类检索能力，遵循"全局导航、候选定位、关联扩展、原文核查、视觉理解"的层次查询逻辑，生成包含回答文本与证据摘要的草稿回答。强制输出证据摘要约束生成不得凭空推断，同时为验证 Agent 提供明确核查靶点。

在验证 Agent 四维独立核查方面，验证 Agent 以刻意区分的检索关键词对草稿回答进行独立交叉核查，从可回答性、证据充分性、准确性、完整性四个维度逐级审核，体现"能否回答，回答是否正确，回答是否完整"的递进逻辑。验证未通过时，输出包含具体问题描述与问题类型标签的结构化反馈。

在迭代反馈重试方面，调度逻辑将结构化反馈注入重试提示，引导检索 Agent 以差异化关键词定向重检索，保留已验证正确部分。循环最多执行 $M=2$ 次；若仍未通过，以低置信度标记输出当前最优回答。


### 3.2 开题报告中所列关键问题的解决情况

开题报告提出了三项关键技术难点。由于研究过程中技术路线由知识图谱表示学习（嵌入方法）调整为基于大语言模型的 RAG 方法，本节说明原定关键问题在当前方法体系下的对应解决方案，以及新技术路线额外发现并处理的新问题。

问题一是多模态数据融合与表示，即如何将文本、图像等不同模态数据有效融合并在同一表示空间进行比较和关联。当前采用 MinerU 对产品 PDF 进行高精度多模态解析，统一提取文本块、图片、表格与公式，对非文本内容以视觉语言模型（Qwen-VL）进行语义编码，将视觉内容转化为语义描述与文本信息统一索引，纳入知识图谱构建与检索流程。该方案无需专门的多模态对齐训练，即可实现文本与视觉内容在同一知识图谱中的联合表示。

问题二是知识图谱构建，即如何从大量多模态数据中提取有意义的信息，构建包含产品特性、技术规范等的结构化知识图谱。该问题由 PRAG 方法解决，技能库封装领域先验知识，DKEAgent 在基础图谱上执行全局结构化抽取，将各章节散布的产品-组件-功能-参数层次结构完整聚合，语义实体对齐机制确保新知识准确融合至已有实体。实验结果显示，增强图谱在核心产品节点处关联度提升逾 100%，为问答检索提供了结构化的知识路径。PRAG 还解决了开题报告未预见的跨页面信息断层问题，其全局抽取机制绕过了传统分块级抽取因上下文窗口局限无法跨越页面边界的根本限制。

问题三是问答系统设计，即如何设计基于知识图谱的问答系统，使其能够理解用户提问意图并生成准确回答。该问题由 A-PRAG 方法解决，问题分类机制解决了不同类型问题检索策略不适配的问题，检索 Agent 的分层知识检索实现了从语义意图到图谱精确路径的导航，验证 Agent 的四维独立核查与结构化反馈解决了开环生成中幻觉难以自动发现和纠正的问题，迭代反馈重试机制将整个流程升级为具备自我纠错能力的闭环系统。此外，实践中还发现了不同类型问题的检索行为异质性问题，事实型、计数型、视觉型问题对检索行为的要求本质不同，A-PRAG 的自适应策略注入在检索入口解决了这一结构性矛盾。


### 3.3 创新性的方法、技术、成果

#### 创新一：基于领域知识驱动的知识图谱增强构建方法（PRAG）

产品说明书中描述同一组件的信息往往散落在不同章节，以一份典型的消费电子说明书为例：产品概述在第1页，电池参数在第15页，电池安全特性在第30页。现有基于RAG的知识图谱构建方法惯常做法是先对文本分块进行局部的实体与关系抽取，再合并各块结果。这种做法存在一个根本性缺陷：缺乏领域先验知识的结构化引导，"产品—组件—功能—参数—属性"等层次关系没有得到显式建模，导致分散在不同分块中的同一组件信息无法被识别并聚合为一条完整的知识条目。

为解决上述问题，本文提出 PRAG，将领域先验知识以"技能"形式结构化定义，驱动智能体在基础图谱 $G_{v1}$ 上开展全局的层次化知识抽取与融合，流程分为领域知识提取与领域图谱融合两个阶段，整体流程如图3-1所示。

> 图3-1 PRAG 整体方案流程（引自论文图3-1）

在技能库设计方面，传统方法往往将领域知识硬编码于提示或模型权重中，难以跨文档、跨领域复用。本文受智能体技能库范式（VOYAGER [20]、SkillsBench [21]）启发，将领域先验知识以声明式文件分层封装为自描述的技能单元（文件结构见图3-2，详细说明见3.1节）。每个技能单元面向特定领域，描述该领域应当抽取什么知识、按什么顺序、以何种约束输出，而不与任何具体文档或执行代码绑定。这一设计的关键创新在于将领域知识从执行逻辑中完全解耦，DKEAgent 在运行时动态识别文档领域并激活匹配技能，使领域覆盖范围可随技能库扩充持续增长，无需修改底层执行逻辑。

> 图3-2 技能目录结构示例（引自论文图3-3）

在领域知识图谱融合方面，领域知识集 $I_D$ 经两阶段融合写入基础图谱（融合流程见图3-3）。知识映射阶段将五个概念域（产品、组件、功能、参数、属性）转换为图谱节点与有向边，形成以产品节点为根的层次化图结构。语义实体对齐阶段对每个待写入节点构造语义查询，在基础图谱实体向量空间中检索最近邻，余弦相似度达到阈值 $\tau$ 时归并至已有节点，否则创建新节点，最终形成增强图谱 $G_{v2}$。

#### 创新二：基于迭代验证反馈的自适应多智能体检索增强生成方法（A-PRAG）

PRAG 解决了图谱构建层面的跨页面信息断层问题。但在对两个数据集做进一步误差分析时，发现问题的主要来源转移到了检索与生成环节。现有 RAG 系统的"先检索、再生成"单轮开环范式在处理多模态长产品文档时存在三方面不足。首先，检索策略缺乏对问题类型的感知，事实型、计数型、视觉型等不同问题在检索需求上差异显著，统一策略顾此失彼。其次，回答生成没有事实核查环节，模型在检索结果不完整时容易用参数知识"补全"推测，产生幻觉。最后，单次检索没有自我纠错机制，若未命中关键信息，系统只能原样输出，无法发现和修正。

针对上述问题，本文在 PRAG 框架基础上提出 A-PRAG，将单轮开环流水线替换为由代码编排器统一调度的多智能体闭环架构。检索 Agent 在类型策略引导下执行分层知识检索并生成草稿回答，验证 Agent 通过差异化检索路径进行独立事实核查，发现问题时生成结构化反馈驱动重检索。两类 Agent 交替协作，配合确定性的问题分类、策略注入与重试触发逻辑，构建迭代验证反馈（Retrieve-Verify-Refine）闭环范式，整体流程如图3-4所示。

> 图3-4 A-PRAG 整体方案流程（引自论文图4-1）

在自适应检索 Agent 方面，检索 Agent 是闭环中负责知识获取的核心组件，其架构与查询处理流程如图3-5所示。系统首先通过大语言模型将用户问题映射至五类预定义类型之一（事实型、计数型、视觉型、列举型、不可回答型），再从策略仓库中匹配对应的检索策略注入 Agent 的决策空间。事实型约束优先精确文本检索并以原文核查具体数值；计数型强制执行跨页面逐页枚举扫描，禁止依赖摘要统计；视觉型强制调用视觉语义理解，禁止无图作答；列举型要求多页扫描保证完整性；不可回答型要求充分多路检索后方可判定。检索 Agent 完成多轮知识查询后，生成包含回答文本与证据摘要的草稿回答，证据摘要约束生成不得凭空推断，同时为验证 Agent 提供明确核查靶点。验证未通过时，结构化反馈引导检索 Agent 以差异化关键词定向重检索，直至验证通过或达到最大重试次数。

## 5. 尚存的问题及措施

### 5.1 论文后期工作存在的困难和问题

当前主要存在两方面不足。在领域泛化性上，知识图谱增强构建方法目前仅在消费电子产品领域进行了验证，技能库的概念域设计亦以该领域为主要参照。迁移至医疗、化工、制造等其他行业时，技能库需要重新设计领域Schema，适配难度与效果尚未经过测试，这在一定程度上限制了结论的普适性。在评测规模上，MMLongBench-Doc Guidebooks 子集仅包含 196 个问答对，对部分细粒度结论（如不同问题类型间的准确率差异）的统计支撑不够充分，后期将在论文讨论部分明确说明这些结论的适用范围。

### 5.2 如期完成全部论文工作的可能性

从目前进度来看，按期完成全部论文工作是可行的。两个核心方法均已完成设计、实现与实验验证，对比实验与消融实验数据完整，不存在需要重新设计或补充实验的重大遗留问题。后期工作主要为论文修改、格式规范化与答辩准备，性质明确，各阶段进展可控。
