# 第一章 绪论

## 1.1 课题研究背景

### 1.1.1 产品质量关键因素挖掘的现实需求

在当前制造业转型升级与消费电子产品快速迭代的背景下，产品质量管理已成为企业核心竞争力的重要组成部分。产品质量的好坏不仅直接影响用户体验和市场口碑，也关乎企业的研发决策、供应链管理和合规审查等核心业务流程。与此同时，现代产品在设计与制造过程中积累了大量的技术文档，涵盖产品说明书、技术白皮书、产品规格文档、操作指南等多种形式，这些文档蕴含着关于产品组件构成、功能特性、性能参数、安全属性等丰富的产品质量关键信息。

然而，传统的产品质量管理方法在面对上述文档时存在明显局限。基于统计方法的质量分析依赖于对已知关键因素的量化建模，难以从非结构化的文档内容中自动发现潜在的质量影响因素；基于领域专家经验的人工分析方式，不仅效率低下、难以规模化，还容易受到主观判断的干扰，导致结论缺乏系统性和可复现性。面对页数动辄数十页、内容涵盖多个专业领域的长篇产品文档，人工逐一阅读、归纳和关联分析的成本极为高昂。因此，如何借助自然语言处理与知识工程技术，从多模态产品文档中自动化地挖掘和结构化产品质量关键因素，已成为产业界与学术界共同关注的重要研究方向。

本课题来源于学校与企业之间的合作项目，旨在面向消费电子领域的产品质量管理需求，研究基于知识驱动的产品质量关键因素自动挖掘方法，为企业在产品设计评审、质量基线建立和用户支持等场景中提供智能化的知识检索与问答支持。

### 1.1.2 多模态产品文档带来的技术挑战

产品文档具有鲜明的多模态、长文档特征，这为自动化信息挖掘带来了独特的技术挑战。

**文档内容的多模态性。** 产品说明书和技术白皮书通常以PDF格式呈现，其中包含丰富的文本描述、参数规格表格、硬件组件示意图、操作步骤插图以及注意事项图标等多种内容形式。这些不同模态的内容在语义上紧密关联——例如，某个硬件组件的参数可能以图片形式嵌入表格，而其操作说明则以文字形式分布在另一章节。单一依赖文本处理的方法无法覆盖这部分视觉信息，进而导致知识抽取的遗漏。

**产品信息的跨页面分散性。** 产品说明书为了便于用户查阅，通常按照功能模块分章节组织内容。以一份智能手表说明书为例，产品概述位于第1页，电池参数散布于第15页，电池安全特性则在第30页另行说明。关于同一产品组件的完整信息往往跨越多个章节和页面，任何单一页面的内容都无法完整描述该组件的全貌。这种信息分散的特点，使得依赖局部上下文的方法在构建完整产品知识表示时面临根本性困难。

**产品知识的层次化结构性。** 产品知识具有天然的层次化组织特征，遵循"产品包含组件、组件具有属性、组件实现功能、功能拥有参数"的层次语义结构。这种层次关系对于回答产品质量相关问题至关重要——例如，要回答"该产品血压测量功能涉及哪些硬件"，系统需要理解功能与组件之间的归属关系，而不仅仅是定位到包含"血压"关键词的文本段落。通用的实体关系抽取方法缺乏对这类产品领域先验结构的建模能力，难以自动形成完整的产品知识体系。

### 1.1.3 知识图谱与大语言模型技术的机遇

近年来，知识图谱与大语言模型（Large Language Model, LLM）的快速发展为上述问题的解决提供了新的技术路径。

知识图谱以图结构组织和表示领域知识，能够显式地建模实体之间的语义关系，天然适合表达"产品-组件-功能-参数"这类层次化的产品知识体系。基于图结构的知识检索不仅能够实现精确的实体定位，还能通过关系路径进行多跳推理，支持复杂的产品质量因素关联分析。

与此同时，LLM在自然语言理解和生成方面展现出强大的通用能力。以GPT系列、Qwen系列为代表的大语言模型，不仅能够理解复杂的用户提问语义，还能在给定上下文的条件下生成流畅、准确的回答，为构建端到端的产品文档问答系统提供了基础。视觉语言模型（Vision-Language Model, VLM）的兴起则进一步打通了图文理解的通道，使系统能够对产品文档中的图表、示意图等视觉内容进行语义分析。

检索增强生成（Retrieval-Augmented Generation, RAG）范式将知识图谱的结构化检索能力与LLM的生成能力相结合，成为构建开放域知识问答系统的主流框架。LightRAG、GraphRAG等图结构RAG方法的出现，以及RAG-Anything等面向多模态文档的统一RAG框架的提出，使得从复杂文档中构建知识图谱并支持多模态问答成为可行。然而，现有RAG框架在处理产品文档这一特定领域时仍面临若干未解决的核心问题：基于文本分块的图谱构建方式无法捕获跨页面的产品级结构化知识，单次"检索-生成"管道在面对不同类型的产品质量问题时缺乏足够的策略适应性，且生成阶段缺乏对事实正确性的独立验证机制。这些问题共同制约了现有方法在产品质量关键因素问答场景下的准确率上限。

### 1.1.4 本文的研究定位

基于上述背景，本文面向多模态产品文档中产品质量关键因素的自动挖掘与问答这一核心任务，在RAG-Anything多模态统一RAG框架基础上，从知识存储与知识检索生成两个层面提出针对性的改进方法，构建PRAG（Product Retrieval-Augmented Generation）框架。在知识存储层面，本文提出基于领域模型驱动的产品知识图谱增强构建方法，通过引入产品领域先验知识和确定性知识抽取流水线，将分散于长文档各处的产品级结构化信息系统性地融合进知识图谱；在知识检索与生成层面，本文进一步提出基于迭代验证反馈的自适应多智能体检索增强生成方法，以问题类型自适应的检索策略和多智能体"检索-验证-修正"闭环机制，提升系统在面对复杂多样的产品质量问题时的回答准确性与事实可靠性。

## 1.2 本文的研究思路与结构安排

### 1.2.1 研究思路

本文的整体研究思路以"发现问题—分析根因—分层解决—系统验证"为主线展开。

首先，通过分析现有RAG方法在产品文档问答任务上的局限性，本文将核心问题归结为两个相互独立但彼此关联的层次：其一是**知识表示层**的不完整性——现有分块级图谱构建方式无法捕获产品文档中的跨页面远距离语义关联，导致产品级结构化知识大量缺失；其二是**检索生成层**的不适应性——单次"检索-生成"管道缺乏对问题类型的策略差异化支持，且不具备独立的事实验证与自我纠错能力。

针对知识表示层的问题，本文引入产品领域先验知识（Product Schema），将其作为知识抽取的结构化约束模板，并设计确定性知识抽取流水线（DKE Pipeline）代替LLM进行任务编排。这一设计将"发现哪些产品实体存在"与"抽取单一实体的完整详情"两个子任务解耦，前者由发现子Agent在全局范围内枚举，后者由详情抽取子Agent独立、并行地完成，最终以程序逻辑合并结果，避免了LLM批量输出时的质量退化。

针对检索生成层的问题，本文引入问题类型感知机制，对不同类型问题（事实型、计数型、视觉型、列举型、不可回答型）分别注入差异化的检索策略，使检索Agent的行为模式与问题实际需求相匹配。同时，引入独立的验证Agent，通过差异化检索路径对草稿回答进行交叉核查，并将发现的具体问题转化为结构化反馈驱动检索Agent进行定向修正，将整个问答流程从开环管道升级为具备自我纠错能力的闭环系统。

两套方法在系统层面形成有机互补：增强图谱提供的产品层次化结构知识，既是检索Agent的先验导航依据，也为验证Agent的交叉核查提供了更丰富的备选检索入口；而闭环验证机制则反过来确保了从增强图谱中检索到的知识得以被充分、准确地转化为最终回答。

### 1.2.2 论文结构安排

本文共分五章，各章内容安排如下：

**第一章 绪论。** 本章介绍课题的研究背景，分析多模态产品文档中产品质量关键因素挖掘所面临的现实需求与技术挑战，阐述知识图谱与大语言模型技术为解决上述问题提供的机遇，明确本文的研究定位，并介绍全文的研究思路与章节结构安排。

**第二章 国内外研究现状与相关工作。** 本章对与本文研究密切相关的领域进行系统综述，包括：知识图谱构建与表示学习的发展脉络，检索增强生成方法的主要范式与进展，面向多模态文档的RAG框架现状，以及多智能体系统在知识检索与验证任务中的应用。在此基础上，梳理现有方法的局限性，明确本文研究的出发点。

**第三章 基于领域模型驱动的产品知识图谱增强构建方法。** 本章提出PRAG框架中的知识存储增强模块。首先，形式化描述传统分块级图谱构建在产品文档场景下的问题；然后，详细介绍产品领域模型的设计、DKE Pipeline的架构与五阶段工作流、任务特化子Agent的配置机制，以及基于语义实体对齐的图谱融合方案；最后，在两个多模态产品文档问答数据集上开展对比实验与案例分析，验证所提方法的有效性。

**第四章 基于迭代验证反馈的自适应多智能体检索增强生成方法。** 本章提出PRAG框架中的知识检索与生成优化模块。首先，分析传统单次"检索-生成"范式在产品质量问答任务中的三类局限；然后，详细介绍A-PRAG的整体架构——"检索-验证-修正"闭环流程设计、问题类型分类与自适应策略注入机制、自适应检索Agent与验证Agent的协作设计，以及A-PRAG与PRAG增强图谱的系统协同方式；最后，在与第三章相同的数据集上开展对比实验、消融实验与案例分析，验证各设计模块的有效性及其协同增益。

**第五章 结论。** 本章对全文的研究工作进行总结，归纳本文在知识存储增强与知识检索生成优化两个层面的主要贡献，并针对当前方法的局限性，对未来可能的研究方向进行展望。
