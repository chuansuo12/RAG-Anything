## P2  课题来源与背景（~1min）

**文本**
- 课题来源：学校与企业合作项目，面向工业产品制造领域的产品质量管理需求
- 产品文档特征：PDF 多模态（文本、表格、示意图）、长文档、信息跨页面分散
- 产品知识的 1-N-N 层次结构：产品→组件→功能→参数→属性
- 现有方法的两个核心不足：①远距离领域关系难以构建 ②单轮生成缺乏可靠性保障

**图片** 截取一份产品说明书的典型页面（如 HUAWEI WATCH D 说明书封面 + 内页参数表 + 安全说明页），展示"同一组件信息散落不同章节"

**讲述要点** 先讲企业痛点（文档多、人工读不过来），再用说明书实例直观说明信息跨页面分散的问题，最后归纳为两个技术挑战。

## P10  创新二 A-PRAG 检索与验证智能体（~40s）

**文本**
- 检索智能体：五类问题分类 → 策略注入 → 分层检索 → 草稿回答 + 证据摘要
- 验证智能体：四维核查（可回答性→证据充分性→准确性→完整性）
- 未通过 → 结构化反馈 → 定向重检索，最多 M=2 次
- 控制逻辑以程序实现，智能体仅在检索推理环节保留自主决策

**图片** 论文图 4-4（检索智能体架构）+ 论文图 4-5（验证智能体架构），可并排放置

**讲述要点** 分别讲检索和验证两个智能体的核心机制，强调程序编排与 LLM 推理的分工。

## P11  对比实验结果（~40s）

**文本**
- 五种方法在两个数据集上的准确率对比表：

| 方法 | MMLongBench-Doc (%) | MPMQA (%) | 平均 (%) |
|------|---------------------|-----------|----------|
| LightRAG | 31.4 | 29.8 | 30.6 |
| MMRAG | 35.7 | 34.1 | 34.9 |
| RAG-Anything | 41.2 | 39.6 | 40.4 |
| PRAG | 43.2 | 41.4 | 42.3 |
| A-PRAG | 48.1 | 46.3 | 47.2 |

- A-PRAG 较 RAG-Anything 累计提升 6.8pp（相对 16.8%）
- 不可回答问题拒答准确率：A-PRAG 48.6%，唯一不低于整体准确率的方法（差值 +0.5pp）；其余方法均下降 7~9pp

**图片** 对比实验表格（重绘为 PPT 表格）+ 柱状图（各方法准确率对比）

**讲述要点** 先报主实验数字，强调 A-PRAG 在两个数据集上均最优；重点提不可回答问题的幻觉抑制效果，这是区分度最大的指标。

## P12  消融实验结果（~40s）

**文本**
- 消融实验表格：

| 配置 | 平均准确率 (%) | 变化 |
|------|--------------|------|
| A-PRAG（完整） | 47.2 | — |
| w/o 验证智能体 | 41.6 | −5.6 |
| w/o 增强图谱 | 43.6 | −3.6 |
| w/o 产品结构化信息 | 44.8 | −2.4 |
| w/o 问题分类 | 45.5 | −1.7 |
| w/o 重试机制 | 45.9 | −1.3 |

- 贡献层次：验证智能体（−5.6）> 增强图谱（−3.6）> 产品信息（−2.4）> 分类（−1.7）> 重试（−1.3）
- 关键发现：移除验证智能体后（41.6%）低于 PRAG 基线（42.3%），验证智能体是超越基线的关键
- 增强图谱与闭环架构相互促进，非简单叠加

**图片** 消融实验表格 + 各配置准确率降幅的条形图

**讲述要点** 用贡献层次排序快速讲清各模块重要性，重点强调验证智能体和增强图谱的协同关系。

## P13  案例分析（~40s）

**文本**
- 案例一（计数型）：问"该笔记本右侧有几个接口？"（答案 3 个）
  - RAG-Anything：语义检索混入左侧接口 → 错答 5 个
  - A-PRAG：计数型策略 → 强制逐页枚举 → 正确回答 3 个，验证智能体交叉核查确认
- 案例二（不可回答）：问"屏幕低亮度频闪是哪里出了问题？"（文档无此信息）
  - PRAG：检索到屏幕参数后臆造"PWM 调光频率过低"→ 幻觉
  - A-PRAG：多关键词充分检索无命中 → 验证智能体判定不可回答 → 正确拒答

**图片** 两个案例的对比展示（左：基线错误回答摘要，右：A-PRAG 正确回答摘要），可用文本框对比排版

**讲述要点** 每个案例 20 秒，先说问题和正确答案，再对比基线为什么错、A-PRAG 为什么对。案例一体现自适应策略的价值，案例二体现验证智能体的幻觉抑制能力。
