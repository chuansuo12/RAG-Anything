# 第五章 结论

## 5.1 研究工作总结

本文针对现有RAG方法在处理多模态产品长文档时面临的产品级结构化知识缺失、检索策略缺乏问题适应性以及生成阶段缺乏事实验证机制等问题，在RAG-Anything框架基础上提出了A-PRAG框架，从知识存储与知识检索生成两个层面协同提升产品质量关键因素问答的准确性。

在知识存储层面，本文提出基于领域模型驱动的产品知识图谱增强构建方法。通过引入预定义的产品领域模型（Product Schema）规定"产品-组件-功能-参数-属性"五层概念域结构，驱动确定性知识抽取流水线（DKE Pipeline）在基础图谱上执行全局产品信息抽取。DKE Pipeline将编排逻辑从LLM中剥离，由程序逻辑控制"概要抽取—实体发现—详情并行抽取—确定性合并"的五阶段工作流，各任务特化子Agent独立完成单一实体级别的全局检索与结构化抽取；抽取结果经语义实体对齐机制融合回图谱，形成具备完整层次结构的增强图谱。实验表明，PRAG平均准确率达到42.3%，较RAG-Anything基线提升1.9个百分点。

在知识检索与生成层面，本文提出基于迭代验证反馈的自适应多智能体检索架构A-PRAG。系统以代码编排器为调度核心，实现问题类型分类（事实型、计数型、视觉型、列举型、不可回答型）、类型特定检索策略注入、验证判断与重试触发的确定性调度；自适应检索Agent通过分层检索生成有据可查的草稿回答；独立验证Agent从证据充分性、完整性、可回答性与准确性四个维度进行交叉核查，将发现的问题转化为结构化反馈驱动检索Agent定向修正。A-PRAG平均准确率达到47.2%，较PRAG提升4.9个百分点，较RAG-Anything累计提升6.8个百分点。消融实验表明，两个方法模块在系统层面存在强互补依赖关系——增强图谱为闭环检索提供结构化先验导航，闭环验证机制则确保图谱知识得以被充分、准确地转化为最终回答。

## 5.2 研究局限性与未来展望

本文方法仍存在若干局限性。其一，产品领域模型以消费电子产品为主要参照，迁移至化工、制药等其他行业时，概念域定义需要领域专家介入进行实质性调整，开箱即用性有限。其二，DKE Pipeline的抽取质量依赖基础图谱及其向量索引的完整性，若底层文档解析存在缺失，抽取质量将受到相应影响。其三，A-PRAG的闭环架构在最坏情况下需要多次Agent调用，推理开销显著高于单次管道，在对延迟敏感的在线服务场景中存在部署成本压力。此外，本文实验以英文消费电子文档为主，结论向其他语言和产品类型的外推有待进一步验证。

针对上述局限性，未来研究可从以下方向展开：探索基于文档内容的领域模型自动归纳方法，降低新领域迁移的人工成本；研究基础图谱构建与领域增强的端到端联合优化方案，从源头减少产品信息碎片化；针对推理开销问题，研究基于问题难度的自适应验证深度策略，仅对高风险问题执行完整闭环，以平衡性能与效率；在更长远的方向上，面向多版本产品文档的增量知识图谱更新机制与跨文档的产品知识对比分析，亦具有重要的实践价值。
