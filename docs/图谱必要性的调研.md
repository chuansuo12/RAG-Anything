有，但要先说结论：**目前没有一篇“最终定论式”的论文，能证明 Agentic 一定优于图谱，或图谱一定优于 Agentic。**更准确的情况是，近两年的研究开始做**局部、任务级**的对比；结果通常是：**Agentic 在某些多跳检索任务上很强，图谱在结构约束、可解释性和稳定性上更占优**，而不同论文的结论还会受数据集、模型、工具接口和评测指标影响。([arXiv][1])

能直接支持“Agentic 可能强于传统图谱式方案”的一篇是 **A-RAG**。这篇工作把 Agentic RAG 做成分层检索接口，并在实验里**把 GraphRAG 和 Workflow RAG 一起纳入比较**；作者明确写到，A-RAG（尤其 Full 版本）在多个基准上优于现有 Graph-RAG 和 Workflow RAG 方法。它还指出，很多现有 Graph-RAG / Workflow RAG 方法并不能稳定超过简单基线。([arXiv][2])

但反过来，也不能据此推出“图谱不重要”。**Graph-R1** 是一篇把图谱与 Agentic 结合的工作，它把检索建模为多轮 agent-environment interaction，并报告说自己在标准 RAG 数据集上优于传统 GraphRAG 和 RL-enhanced RAG，体现在 reasoning accuracy、retrieval efficiency 和 generation quality 上。这个结果说明：**Agentic 和图谱不是二选一，很多时候是叠加后才更强**。([arXiv][3])

如果你想要“客观论证”里最有力的一类证据，我建议优先看两种论文：
第一类是**带统一基准的比较论文**，像 A-RAG 这种直接把 GraphRAG 放进实验矩阵；第二类是**诊断型 benchmark**，像 AgenticRAGTracer，它专门做 step-by-step 验证，结果显示即使最强模型，在 agentic multi-hop 检索里也常出现 reasoning chain 扭曲、过早收缩或过度延展等失败模式。这个证据很重要，因为它说明 Agentic 并不是天然“替代图谱”——它只是把推理过程转成了另一种可失败的搜索过程。([arXiv][4])

所以，如果你要写论文或做方案论证，可以比较稳妥地表述为：**现有证据支持“Agentic 能在部分多跳任务上接近或超过图谱式 RAG”，但尚无普适结论表明它可以完全替代图谱；更合理的方向是 Agentic + Graph 的混合范式。** 这也和 Agentic RAG survey 对领域演化的归纳一致：它把 Graph RAG 视作相关谱系之一，而不是被完全淘汰的旧范式。([arXiv][5])
